Sistema Operativo e Avvio

Sistema Operativo

Il sistema operativo (SO) è un software fondamentale che gestisce le risorse hardware e software di un computer. Tra le sue principali funzioni ci sono:

  • Gestione delle risorse hardware come CPU, memoria, dispositivi di input/output.
  • Fornitura di un'interfaccia per gli utenti e le applicazioni per accedere alle risorse del sistema.
  • Gestione dei processi e della loro esecuzione in modo multitasking.
  • Gestione dei file e delle operazioni di input/output sul disco rigido.
  • Fornitura di servizi di rete come la connettività Internet e la gestione dei protocolli di comunicazione.
  • Garanzia della sicurezza e della protezione dei dati.

Avvio del Sistema Operativo

Il processo di avvio del sistema operativo inizia quando il computer viene acceso. Durante l'avvio, il computer esegue una sequenza di operazioni per preparare il sistema operativo per l'uso. Queste operazioni includono:

  • Il BIOS o l'UEFI effettua i test hardware iniziali e carica il bootloader dal disco rigido o da altri dispositivi di avvio.
  • Il bootloader, come GRUB per Linux o NTLDR per Windows, carica il sistema operativo in memoria e ne avvia l'esecuzione.
  • Il sistema operativo completa l'avvio, inizializzando i driver di dispositivo, avviando i servizi di sistema e presentando l'interfaccia utente all'utente.

Tipologie di Interfacce

Le interfacce utente consentono agli utenti di interagire con il computer e le sue applicazioni. Le principali tipologie di interfacce includono:

  • Interfaccia Utente: L'interfaccia utente permette agli utenti di interagire con il computer attraverso comandi e testo. Ad esempio, una shell di linea di comando.
  • Interfaccia Grafica (GUI): L'interfaccia grafica consente agli utenti di interagire con il computer attraverso elementi visivi come finestre, icone e menu.

Risorse File e Cartelle

I file e le cartelle sono le unità di base per l'organizzazione e la gestione dei dati in un computer. Un file è una raccolta di dati con un nome associato, mentre una cartella è una raccolta di file e altre cartelle. Le operazioni principali sui file e le cartelle includono:

  • Creazione, modifica e eliminazione di file e cartelle.
  • Accesso e lettura dei contenuti dei file.
  • Spostamento e rinomina di file e cartelle.
  • Condivisione e protezione dei file attraverso i permessi di accesso.

Hard Disk e File System

L'hard disk è un dispositivo di memorizzazione permanente che conserva i dati anche quando il computer è spento. È diviso in una o più partizioni, ognuna delle quali è formattata con un file system. I file system più comuni includono NTFS per Windows, HFS+ per macOS e ext4 per Linux. Il file system gestisce la struttura e l'accesso ai file e alle cartelle sul disco rigido.

Windows, macOS e Linux a Confronto

I principali sistemi operativi per computer sono Windows, macOS e Linux. Ognuno ha caratteristiche uniche e differenze significative:

  • Windows: Sviluppato da Microsoft, Windows è noto per la sua ampia compatibilità con il software e l'hardware di terze parti. Offre un'interfaccia grafica user-friendly e un supporto per giochi e applicazioni multimediali.
  • macOS: Il sistema operativo di Apple è progettato per i computer Mac. È noto per la sua stabilità, la sua integrazione con l'ecosistema Apple e le sue funzionalità creative come iMovie e GarageBand.
  • Linux: Basato sul kernel Linux, Linux è un sistema operativo open source utilizzato principalmente nei server e nei computer desktop degli utenti avanzati. È altamente personalizzabile e offre una vasta gamma di distribuzioni come Ubuntu, Fedora e Debian.

Domande

  1. Quali compiti svolge il programma boot loader?
    Il boot loader ha il compito di avviare il sistema operativo, leggendo il Master Boot Record (MBR) dal disco rigido e avviando il sistema operativo principale.
  2. Qual è il formato del Master Boot Record?
    Il Master Boot Record è una struttura di 512 byte che contiene un programma di avvio (boot loader) e una tabella delle partizioni.
  3. Spiega che cos'è e come viene usata la Partition Table.
    La Partition Table è una tabella situata nel MBR che contiene informazioni sulle partizioni presenti sul disco rigido. Viene utilizzata dal sistema operativo per individuare le partizioni e avviare il sistema operativo dalla partizione attiva.
  4. Descrivi il formato di una entry della Partition Table.
    Ogni entry della Partition Table contiene informazioni come l'indirizzo di partenza e la dimensione della partizione.
  5. Che cosa significa che una partizione è attiva?
    Una partizione attiva è quella che contiene il sistema operativo avviabile. Il boot loader avvia il sistema operativo dalla partizione attiva.
  6. Che differenza c'è tra le notazioni CHS e LBA con cui si indirizzano i dati sul disco?
    CHS (Cylinder-Head-Sector) è un metodo di indirizzamento basato sulla geometria del disco, mentre LBA (Logical Block Addressing) indirizza i settori in modo logico.
  7. Come si interpreta un dato che occupa più byte secondo la tecnica little-endian?
    In Little-Endian, i byte meno significativi vengono memorizzati per primi. Questo significa che l'ordine dei byte è invertito rispetto all'ordine di scrittura tradizionale.
  8. Confronta le tecniche del multi-boot e della virtualizzazione dei Sistemi Operativi.
    Il multi-boot consente di avviare più sistemi operativi su un singolo computer, mentre la virtualizzazione emula un intero ambiente hardware per eseguire un sistema operativo all'interno di un altro sistema operativo.
